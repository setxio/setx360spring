# SETX.io & SETX 360: Supplementary Knowledge Base & Business Logic Constraints

## 1. Local Geographic & Infrastructure Constraints
Antigravity must understand the regional focus of the pilot to properly structure database seeding, multi-tenant grouping, and edge latency routing.

- **Primary Jurisdictions**: The initial deployment is strictly isolated to Jefferson County and Orange County in Southeast Texas.
- **Target Cities for Local Routing**: Data maps, local delivery nodes, and municipal tools must prioritize localized routing for cities including Groves, Beaumont, Port Arthur, Orange, Nederland, and Port Neches.
- **Database Seeding Constraint**: When generating mock data or testing geolocation endpoints, the AI must explicitly use Southeast Texas boundaries rather than generic default coordinates (e.g., San Francisco or Austin).

## 2. Low-Capital Capitalization & Bootstrap Rules
Because the platform is launching on a lean, highly efficient development model, the code architecture must enforce extreme capital efficiency.

- **The 20 XRP Hard Ceiling**: The live master treasury pool will initially be seeded with exactly 20 XRP. The AI must optimize all ledger code to minimize unnecessary account creations, object creations, or high-frequency ledger state modification flags that cost extra drops.
- **Zero-Reserve Multi-Tenancy**: The IDE must treat the `profiles.dest_tag` off-chain mapping as the absolute truth for user balances. It must strictly reject any code patterns that attempt to create individual on-chain r-addresses for standard users, as this would trigger a 1 XRP lockup per resident that violates bootstrap constraints.
- **Self-Replenishing Gas Loop**: The code logic inside the marketplace module must explicitly tie the 1% platform fee collected via Stripe Connect to an automated exchange function that uses the fiat revenue to market-buy XRP, continually pouring it back into the operational county gas wallets.

## 3. Legal Trigger Alignments (Texas HB 4518)
The deployment of specific features inside the code is legally gated by the passing of the Texas Decentralized Unincorporated Nonprofit Association (DUNA) framework.

- **The 100-Member Legal Milestone**: The AI must understand that the DUNA structure requires a minimum of 100 participating members to activate its legal status and liability shield under Texas law.
- **The Social/Civic Onboarding Gate**: To meet this requirement without requiring 100 upfront business partnerships, the application logic must prioritize a frictionless, free-to-use Social Media and Civic Discussion feed.
- **Conduct-Inferred Consent**: The registration flow must treat account creation and participation in the social feed as an implicit signature of the DUNA mutual agreement. The terms must be hardcoded into the user onboarding workflow.

## 4. Software Stack Integration Mapping
The IDE must completely ignore legacy web-builder patterns (such as WordPress, Bubble, or monolithic PHP backends). The entire stack must be built using modern, edge-native architecture:

- **Frontend Ecosystem**: Next.js App Router deployed natively to Vercel. UI styling must feature clean, high-fidelity dark-mode aesthetics, glassmorphism components, and neon-based color gradients.
- **Backend & Storage**: Supabase handles the core relational PostgreSQL data, user sessions via Go/TypeScript libraries, Row-Level Security (RLS), and database webhooks.
- **Payment Infrastructure**: Stripe Connect is the exclusive bridge for fiat handling, multi-vendor payment splitting, automated delivery driver allocations, and the fiat-to-token on-ramp.
